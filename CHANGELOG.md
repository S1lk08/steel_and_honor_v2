# Changelog

All notable changes to Steel & Honor will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2024-12-XX

### Added
- Initial release of Steel & Honor
- Complete kingdom management system with create, rename, and color commands
- Role-based permission system (Leader, Officer, Politician, Military, Citizen)
- Territory claiming system with visual borders
- War system with 15-minute prep phase and 25-minute war duration
- Kill tracking system (excludes civilians and politicians)
- Alliance system for war assistance
- Comprehensive command suite with tab completion
- In-game GUI menu with full tab completion support
- Visual HUD overlays for war status and kingdom entry
- Particle effects for city entry and war declarations
- Xaero's Minimap and World Map integration
- Banner-based kingdom crests
- Smart overlay positioning to prevent overlap
- OPAC claim text hiding for cleaner UI

### Changed
- Default kingdom design changed from patterned banner to plain white
- War timer display shows both prep and active phases separately
- HUD panels optimized for better visibility

### Fixed
- Timer synchronization issues between client and server
- Overlay positioning conflicts between war and kingdom displays
- Tab completion accuracy and performance
