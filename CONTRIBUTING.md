# Contributing to Steel & Honor

Thank you for your interest in contributing to Steel & Honor! This document provides guidelines and instructions for contributing.

## Code of Conduct

- Be respectful and considerate
- Provide constructive feedback
- Help maintain a welcoming environment

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/steel-and-honor.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Submit a pull request

## Development Setup

See the [README.md](README.md#development) for detailed setup instructions.

## Coding Standards

### Kotlin Style

- Follow Kotlin coding conventions
- Use meaningful variable and function names
- Add KDoc comments for public APIs
- Keep functions focused and single-purpose

### Code Structure

- Keep server and client code separated
- Use appropriate access modifiers
- Follow the existing package structure
- Add appropriate error handling

### Commit Messages

Use clear, descriptive commit messages:

```
feat: Add new war victory conditions
fix: Resolve timer synchronization issue
docs: Update README with new features
refactor: Improve kingdom data structure
```

## Pull Request Process

1. Ensure your code compiles without errors
2. Test your changes in both single-player and multiplayer
3. Update documentation if needed
4. Update CHANGELOG.md with your changes
5. Ensure all checks pass
6. Request review from maintainers

## Reporting Issues

When reporting bugs, please include:

- Minecraft version
- Mod version
- Fabric Loader version
- Steps to reproduce
- Expected vs actual behavior
- Relevant logs (if any)

## Feature Requests

Feature requests are welcome! Please:

- Check if the feature already exists
- Explain the use case
- Describe the desired behavior
- Consider implementation complexity

## Questions?

Feel free to open an issue for questions or discussions about the mod.

Thank you for contributing to Steel & Honor!
